# 各阶段测试文件

本目录包含各个开发阶段的测试文件和脚本。

## 目录结构

```
各阶段测试用/
├── README.md                    # 本文件
├── 第一阶段测试/                # 第一阶段测试文件
│   ├── README.md
│   ├── 第一阶段测试说明.md
│   ├── 快速测试指南.md
│   ├── test-phase1.sh
│   └── test-phase1.ps1
└── 第二阶段测试/                # 第二阶段测试文件（待添加）
```

## 使用方法

### 第一阶段测试

进入第一阶段测试目录：
```bash
cd 第一阶段测试
```

然后选择以下方式之一：

#### 方法1：快速测试（推荐）
查看 `快速测试指南.md` 进行5分钟快速测试。

#### 方法2：详细测试
查看 `第一阶段测试说明.md` 了解详细的测试步骤。

#### 方法3：自动化测试

**Windows (PowerShell)**
```powershell
cd 第一阶段测试
.\test-phase1.ps1
```

**Linux/Mac (Bash)**
```bash
cd 第一阶段测试
chmod +x test-phase1.sh
./test-phase1.sh
```

## 测试内容

### 第一阶段测试包括：

1. **服务状态检查**
   - 后端服务器 (端口 3000)
   - 前端服务器 (端口 5173)
   - PostgreSQL (端口 5432)
   - Redis (端口 6379)

2. **后端接口测试**
   - 健康检查接口 (`/health`)
   - 数据库连接测试 (`/api/test/db`)
   - Redis连接测试 (`/api/test/redis`)

3. **代码质量检查**
   - 后端代码 lint 检查
   - 前端代码 lint 检查

4. **Docker容器检查**
   - PostgreSQL 容器状态
   - Redis 容器状态

5. **WebSocket测试**
   - 访问 http://localhost:5173/test-websocket
   - 检查连接状态和消息收发

## 注意事项

- 运行测试前，确保已启动所有必要的服务
- 确保已安装所有依赖 (`npm install`)
- 确保 Docker 容器正在运行
- 确保环境变量配置正确

## 测试通过标准

所有测试项都通过后，该阶段才算完成。如果某个测试失败，需要修复问题后重新测试。
